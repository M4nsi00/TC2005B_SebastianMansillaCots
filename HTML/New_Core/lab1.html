Preguntas

1. ¿Cuál es la diferencia entre internet y la World Wide Web?

En este caso la mayor diferencia es que el internet es la infraestructura que conecta a una computadora con otra, la World Wide Web (WWW) es un servicio que recopila archivos usando la infraestructura de internet.

2. ¿Cuáles son las partes de un URL?

Una URL (Uniform Resource Locator) esta compuesta por:

  • Protocolo: (HTTPS)
  • Dominio: Nombre de identificación de una o varias direcciones de IP
  • Path: Es una especie de Guía de localización de lo que buscas en el server
  • Parámetros: Información adicional que filtra información

3. ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

  • GET: Método para pedir datos de una fuente específica, no modifica datos.
  • HEAD: Similar al método GET pero en este caso no regresa un BODY (Cuerpo del archivo), sirve para checar.
  • POST: Método para mandar datos al servidor, necesita un BODY.
  • PUT: Método que cumple la función de editar algo.
  • PATCH: Método para aplicar modificaciones parciales.
  • DELETE: Método para borrar

4. ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas Uniform Resource Locator usuario y contraseña en algún sitio? ¿Por qué?

En este caso se usa POST, ya que la respuesta se manda desde dentro del BODY del archivo.

5. ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

Se usa el método GET, ya que, este método se usa para pedir o obtener información.

6. Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

Esto significa que el servidor web logro de forma exitosa dar respuesta al request que se le hizo, en otras palabras no es un error es lo que se esperaba que respondiera.

7. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

No, este tipo de errores involucra que el servidor no encontró el sitio al que el usuario quiere entrar, podría pasar porque el usuario anotó mal el URL y por ende no existe en el servidor, en este caso no se tiene que arreglar nada.

8. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

Si, esto es porque existe algo entre el servidor y el archivo que causó que el servidor estuviera en una situación que no sabe cómo manejar, es por esto que el desarrollador debe corregirlo porque el problema proviene de su parte.

9. ¿Qué significa que un atributo HTML5 este despreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML4 que en HTML5 estén desaprobados.

Son aquellos atributos que en un futuro serán removidos, en otras palabras son cosas que eventualmente dejarán de tener soporte, y aunque en el momento funcionan uno no debería confiar de ellas si está desarrollando. Algunos ejemplos son: applet, basefont, big, center

10. ¿Cuáles son las diferencias principales entre HTML4 y HTML5?

Entre las principales diferencias esta:

  • Declaración del Doctype simplificado
  • En html4 se usan cosas como div y span, cosas que en html5 ya no son necesarias, haciendo que el código sea más fácil de leer y de entender.
  • En html5 se abre el apoyo a multimedia, dejando de lado la necesidad de plugins.
  • Se introduce canvas en html5 simplificando el uso de elementos interactivos.
  • En html5 salen nuevos tipos de inputs como email o date
  • En html5 se añade mayor integración de APIs y JavaScript

11. ¿Qué componentes de estructura y estilo tiene una tabla?

Los componentes de estructura son:

  • table: Contenedor principal
  • thead: Contenido del encabezado
  • tbody: Cuerpo o contenido principal
  • tfoot: Pie de la tabla
  • tr: Define una fila
  • th: Define un encabezado de celda
  • td: Celda de datos

Componentes de estilo son:

  • border: define las líneas divisoria entre celdas.
  • padding: añade espacio interno entre th y td.
  • text-align: alinea el texto (center,left…)
  • width: Ancho de la tabla
  • height: Alto de la tabla

12. ¿Cuáles son los principales controles de una forma HTML5?

Algunos de los principales controles son:

  • Input Boxes (Un elemento muy versátil que se modifica dependiendo del tipo de input que se va a usar como email, o password)
  • Text Areas
  • Select Boxes
  • Radio Buttons
  • Check Boxes
  • File Uploads

14.1 ¿Cuál es el ciclo de vida de los sistemas de información?

El ciclo de vida de vida de un sistema de información es el proceso que tiene desde el concepto, abarcando las siguientes fases:

  • Planificación
  • Análisis
  • Diseño
  • Implementación
  • Pruebas
  • Instalación o despliegue
  • Uso y mantenimiento

14.2 ¿Cuál es el ciclo de desarrollo de sistemas de información?

El ciclo de desarrollo de sistemas de información involucra a las fases dentro del ciclo de vida que tienen enfoque en desarrollar el sistema, mientras que el ciclo de vida va desde la idea hasta el sistema se deja de usar, el enfoque de este ciclo está en transformar requisitos en un producto funciona.
